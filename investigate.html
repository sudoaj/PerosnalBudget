Purpose: Budget-specific functionality and workflows

  • Composition: Components work together but stay focused
  • Forms: Category-specific input handling
  • Data Flow: Props down, events up pattern

ÔøΩÔ∏è Utility Layer (/src/lib)

Purpose: Pure functions and business logic

  • utils.ts: General helpers (currency, dates, formatting)
  • storage.ts: Adapter pattern for data persistence
  • export-utils.ts: Complex export logic for multiple formats
  • Benefits: Testable, reusable, framework-agnostic

üè™ State Management (/src/store)

Purpose: Global application state with Zustand

  • Simple: No boilerplate like Redux
  • TypeScript: Full type safety
  • Persistence: Automatic localStorage sync
  • Performance: Minimal re-renders, selective subscriptions

ÔøΩ Type Safety (/src/types)

Purpose: Centralized TypeScript definitions

  • Interfaces: All data structures defined
  • Enums: Category types, frequencies, etc.
  • Unions: Specific string literal types
  • Benefits: Compile-time error checking, autocomplete
